## 1. 游戏概述

### 1.1 游戏简介

《逐梦少女》（Dreamer Heroines）是一款快节奏、高难度的2D横板射击游戏。玩家扮演精英战士，在各种战场环境中与敌军作战，通过精准的射击、灵活的走位和战术决策生存下来。

### 1.2 核心体验

- **紧张刺激**: 快节奏战斗，敌人AI具有威胁性
- **技能驱动**: 玩家技术决定成败，而非数值碾压
- **战术深度**: 掩体利用、武器切换、技能配合
- **成长满足**: 角色培养与武器解锁提供长期动力

### 1.3 目标平台

| 平台 | 优先级 | 备注 |
|------|--------|------|
| PC (Windows) | P0 | 主要开发平台 |
| PC (Linux) | P1 | 兼容支持 |
| Web (HTML5) | P2 | 可选导出 |

### 1.4 目标受众

- 核心动作游戏玩家
- 横板射击爱好者
- 追求挑战的硬核玩家

## 3. 核心玩法机制

### 3.1 移动系统

#### 基础移动

| 操作 | 输入 | 效果 |
|------|------|------|
| 左右移动 | A/D 或 方向键 | 水平移动，速度 200-300 px/s |
| 跳跃 | 空格/W | 垂直起跳，高度约 120 px |
| 二段跳 | 空中再次按跳跃 | 额外跳跃一次（需解锁或特定职业） |
| 下蹲 | S 或 下方向键 | 降低受击判定，可滑入低矮通道 |
| 冲刺 | Shift | 快速位移，短暂无敌帧（0.2s），冷却 2s |

#### 高级移动

- **墙跳**: 贴墙时按跳跃键可蹬墙跳跃
- **滑铲**: 冲刺+下蹲，可穿过狭窄空间并攻击低处敌人
- **急停**: 反向按键可快速停止，提升射击精度

### 3.2 射击系统

#### 基础射击

- **瞄准**: 鼠标控制准星，角色朝向跟随准星
- **射击**: 鼠标左键开火
- **换弹**: R键或弹匣耗尽自动换弹
- **武器切换**: 数字键1-4或滚轮切换

#### 射击精度机制

```
精度影响因素:
1. 移动状态: 静止(100%) > 行走(85%) > 冲刺(60%)
2. 连续射击: 每发子弹累积后坐力，短暂停火恢复
3. 武器特性: 不同武器基础精度不同
```

#### 伤害判定

| 部位 | 伤害倍率 | 视觉效果 |
|------|----------|----------|
| 头部 | 2.0x | 爆头特效 |
| 躯干 | 1.0x | 标准命中 |
| 四肢 | 0.75x | 轻微命中 |

### 3.3 掩体系统

#### 掩体类型

| 类型 | 耐久度 | 可破坏 | 战术价值 |
|------|--------|--------|----------|
| 沙袋 | 中等 | 是 | 临时掩护 |
| 混凝土墙 | 高 | 否 | 可靠掩体 |
| 木箱 | 低 | 是 | 快速掩护 |
| 载具残骸 | 高 | 否 | 大型掩体 |

#### 掩体交互

- **贴靠**: 靠近掩体自动贴靠，减少受击面积
- **盲射**: 贴靠时可举枪盲射，精度降低但保持掩护
- **探头**: 按住方向键探头瞄准，精度恢复但暴露

### 3.4 生命与恢复系统

#### 生命值

- **基础HP**: 100点
- **护甲**: 可叠加，优先吸收伤害
- **生命恢复**: 脱离战斗5秒后缓慢恢复（每秒5点）

#### 恢复道具

| 道具 | 效果 | 掉落来源 |
|------|------|----------|
| 医疗包 | 恢复50 HP | 敌人掉落、场景固定点 |
| 护甲包 | 恢复50护甲 | 精英敌人、宝箱 |
| 肾上腺素 | 10秒内攻击速度+30% | 稀有掉落 |

### 3.5 特殊机制

#### 连杀奖励 (Killstreak)

连续击杀敌人无死亡可激活临时增益:

| 连杀数 | 奖励效果 | 持续时间 |
|--------|----------|----------|
| 3杀 | 移动速度+20% | 30秒 |
| 5杀 | 伤害+25% | 30秒 |
| 8杀 | 无限弹药 | 20秒 |
| 10杀 | 伤害免疫 | 10秒 |

*注: 连杀中断则奖励消失*

## 4. 武器系统设计

### 4.1 武器分类

#### 主武器 (Primary)

| 类别 | 定位 | 代表武器 | 特点 |

### 4.2 武器属性详解

#### 属性列表

```yaml
武器属性:
  damage:           # 基础伤害
  fire_rate:        # 射速 (发/分钟)
  accuracy:         # 基础精度 (0-100)
  recoil:           # 后坐力强度
  magazine:         # 弹匣容量
  reload_time:      # 换弹时间 (秒)
  range:            # 有效射程 (像素)
  mobility:         # 持枪移动速度倍率
  penetration:      # 穿透力 (可穿透敌人数)
```

### 5.2 AI行为系统

#### 行为状态机

```
[Idle] → [Alert] → [Combat] → [Search] → [Return]
  ↑                                      ↓
  └──────────────────────────────────────┘
```

#### 状态说明

| 状态 | 触发条件 | 行为 |
|------|----------|------|
| Idle | 无威胁 | 巡逻、待机 |
| Alert | 听到枪声/发现尸体 | 警戒，向声源查看 |
| Combat | 发现玩家 | 攻击、找掩体、呼叫支援 |
| Search | 丢失玩家视野 | 搜索最后已知位置 |
| Return | 长时间未发现 | 返回原位 |

#### 战术行为

| 行为 | 触发条件 | 执行逻辑 |
|------|----------|----------|
| 找掩体 | HP<50%或换弹 | 寻找最近掩体，保持对玩家方向 |
| 侧翼包抄 | 玩家躲在掩体后 | 分兵绕后，形成交叉火力 |
| 集火 | 多人同时发现玩家 | 同时攻击，提高命中 |
| 撤退 | HP<20%且无支援 | 向友军方向逃跑 |
| 呼叫支援 | 发现玩家或伤亡过半 | 生成增援敌人 |

### 5.3 AI难度参数

| 难度 | 反应时间 | 精度 | 伤害倍率 | 额外行为 |
|------|----------|------|----------|----------|
| 简单 | 1.0s | 40% | 0.5x | 无 |
| 普通 | 0.5s | 60% | 1.0x | 基础战术 |
| 困难 | 0.3s | 75% | 1.25x | 侧翼、集火 |
| 噩梦 | 0.15s | 90% | 1.5x | 全部战术 |

## 6. 关卡设计原则

### 6.1 关卡结构

#### 标准关卡流程

```
[起点] → [遭遇区1] → [战斗区1] → [遭遇区2] → [Boss战] → [撤离点]
            ↓              ↓              ↓
        [支线/秘密]   [补给点]      [剧情事件]
```

#### 区域类型

| 区域类型 | 功能 | 设计要点 |
|----------|------|----------|
| 遭遇区 | 小规模战斗 | 2-4个敌人，热身 |
| 战斗区 | 大规模战斗 | 多波次敌人，掩体丰富 |
| 补给点 | 恢复与整备 | 安全区域，固定补给 |
| Boss区 | 首领战 | 特殊机制，阶段设计 |
| 秘密区 | 隐藏奖励 | 需要探索发现 |

### 6.2 关卡主题

| 主题 | 环境特点 | 特殊机制 |
|------|----------|----------|
| 城市废墟 | 建筑残骸，狭窄街道 | 可破坏墙壁 |
| 军事基地 | 开阔地带，掩体少 | 载具战 |
| 丛林 | 茂密植被，视野受限 | 伏击、陷阱 |
| 地下设施 | 封闭空间，多层结构 | 垂直战斗 |
| 雪地 | 开阔视野，白色背景 | 足迹追踪 |

### 6.3 难度曲线

#### 关卡难度参数

```yaml
关卡1 (新手村):
  enemy_count: 15-20
  enemy_types: [突击兵, 冲锋兵]
  boss: 无
  resources: 充足

关卡5 (中期):
  enemy_count: 40-50
  enemy_types: [全部基础类型]
  boss: 轻型载具
  resources: 适中

关卡10 (后期):
  enemy_count: 60-80
  enemy_types: [全部类型]
  boss: 重型机甲
  resources: 紧张
```

### 6.4 关卡目标类型

| 目标类型 | 描述 | 完成条件 |
|----------|------|----------|
| 歼灭 | 消灭所有敌人 | 击杀率100% |
| 生存 | 坚持指定时间 | 时间到且存活 |
| 护送 | 保护NPC到达目标 | NPC存活且到达 |
| 爆破 | 摧毁指定目标 | 目标被摧毁 |
| 撤离 | 到达撤离点 | 玩家到达撤离点 |
| 暗杀 | 击杀特定目标 | 目标死亡 |

## 9. 技术架构

### 9.1 架构概述

本项目采用 **GDScript + C# 混用架构**，充分利用两种语言的优势：

| 语言 | 优势 | 适用场景 |
|------|------|----------|
| **GDScript** | 热重载支持、与引擎深度集成、快速迭代 | UI、关卡逻辑、快速原型 |
| **C#** | 类型安全、性能优异、工具链完善 | 核心计算、数据管理、复杂算法 |

#### 混用架构原则

```
┌─────────────────────────────────────────────────────────────┐
│                        Godot 4.6.1 引擎                      │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────────┐         ┌──────────────┐                  │
│  │   GDScript   │ ←─────→ │     C#       │                  │
│  │   (动态层)    │  信号   │   (计算层)    │                  │
│  └──────────────┘         └──────────────┘                  │
│         ↑                          ↑                        │
│    热重载支持                  类型安全                      │
│    快速迭代                    性能优化                      │
└─────────────────────────────────────────────────────────────┘
```

**分工原则**:
- 热重载需求高的系统 → GDScript
- 类型安全要求高的系统 → C#
- 性能敏感计算 → C#
- 快速原型迭代 → GDScript

#### 跨语言通信方式

| 通信方式 | 方向 | 适用场景 | 示例 |
|----------|------|----------|------|
| **信号 (Signal)** | 双向 | 事件通知、解耦通信 | `player_damaged` 信号 |
| **方法调用** | 双向 | 直接功能调用 | C# 调用 GDScript 的动画方法 |
| **共享资源** | 双向 | 数据共享 | 通过 Resource 文件共享配置 |
| **Autoload 单例** | 双向 | 全局状态管理 | GameManager、EventBus |

#### GDScript 调试

```gdscript
# 使用 print 快速调试
print("Debug: Player health = ", health)

# 使用 push_warning / push_error
push_warning("Health is low: %d" % health)
if health < 0:
    push_error("Health should never be negative!")

# 条件断点
@tool
class_name DebugHelper

static func breakpoint_if(condition: bool, message: String = "") -> void:
    if condition and OS.is_debug_build():
        print("BREAKPOINT: ", message)
        breakpoint
```

#### C# 调试

```csharp
// 使用 GD.Print 输出到 Godot 控制台
GD.Print($"Player position: {Position}");
GD.PrintErr("This is an error!");

// 条件编译调试
#if DEBUG
    GD.Print("Debug build only");
#endif

// 断点
System.Diagnostics.Debugger.Break();
```

#### 混用调试策略

| 问题类型 | 调试方法 |
|----------|----------|
| 信号未触发 | 检查 `Connect()` 是否在 `_Ready()` 中完成 |
| 类型转换失败 | 使用 `Variant` 检查类型后再转换 |
| 方法调用失败 | 确认方法名大小写（GDScript用snake_case） |
| 性能问题 | 使用 Godot Profiler 分析 |
| 内存泄漏 | 检查信号是否正确断开连接 |

#### 性能监控

